# FuelPhysique — Feature Backlog

Priority key:

- חובה לפני השקה — must-have before launch
- חשוב בהמשך — important soon, but not launch-blocking
- רעיון עתידי — future idea

## חובה לפני השקה

- Dedicated handling for anorexia / eating-disorder risk in nutrition generation and chat responses.
- Dedicated handling for overweight / obesity cases in nutrition generation and chat responses.
- Stronger mobile verification of the current workout tracker and nutrition builder flows.
- A lint step or equivalent static validation in the project scripts.

## חשוב בהמשך

- Evidence-strength indicator for AI answers, with clear explanations for stronger / weaker evidence.
- Context-aware “special card” surfacing when the user asks about an existing tool or dedicated flow.
- Better exercise progress explanations that distinguish true progress from simple volume changes.
- More polished route visualization for running history.
- Expanded workout logging UX polish for fast in-gym use.

## רעיון עתידי

- Leaderboards and verification flows for community challenges.
- Personal memory improvements beyond the current settings-based context.
- Deeper subscriptions and access-control tiers after product-market fit is clearer.
- More advanced coach-style prompts for stalled progress, technique questions, and plan adjustments.
- Additional content types for the chat once the product and cost model are stable.
